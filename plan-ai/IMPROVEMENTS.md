# Plan AI Future Improvements and Features

This document tracks high-impact, advanced architectural and product features proposed for the Plan AI platform infrastructure.

## 1. Auto-Architect from Codebase (The "Dawlati" Integration)

Leveraging the highly compressed `.xml` codebase exports via Repomix, the system can automatically ingest large repositories to map and document real-world architecture.

- **Database ER Diagrams:** Generated instantly by having the AI read `schema.prisma`.
- **Microservice Maps:** Generated by mapping the API boundaries across multiple external repositories.
- **Component Trees:** Generated by mapping out how frontend UI components nest together.

## 2. "Click-to-Edit" Precise AI Targeting

Upgrade the Diagram Live Preview to allow surgical, node-specific modifications without conversational ambiguity.

- **Interaction:** Clicking an individual Mermaid node directly opens an inline AI popover.
- **Execution:** AI isolates the exact node ID and alters only that segment, preventing full-diagram syntax breakages when only small tweaks are desired.

## 3. Diagram-to-Code Boilerplate Generator

A reverse-engineering pipeline that translates visual logic into executable starting points.

- **Interaction:** A "Generate Repository" button attached to structural diagrams (like React -> Express -> Postgres).
- **Execution:** AI parses the structural dependencies and automatically scaffolds Dockerfiles, `package.json`, routes, and foundational files, packaged into a downloadable `.zip`.

## 4. Diagram "Time Machine" (Version Diffing)

Implement a robust state-history visualizer for AI generation cycles.

- **Interaction:** A visual timeline slider across the top of the codebase editor.
- **Execution:** Enables dragging back in time to review previous AI iterations (v1 -> v2 -> v3) and immediately visualizes structural "diffs" of what nodes the AI added or removed between steps.

## 5. Transcript-to-Diagram Generator

Bridge the audio/recording infrastructure directly into visual strategy.

- **Interaction:** A "Visualize Conversation" action added to the Transcript UI.
- **Execution:** AI analyzes the structure of long-form audio transcripts (e.g., client requirement meetings, system design interviews) and deterministically outputs a Sequence Diagram or User Journey map documenting exactly what was decided.
